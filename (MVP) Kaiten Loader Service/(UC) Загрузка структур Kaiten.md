# UC Загрузка структур Kaiten

---

## Описание

Пошаговое описание получения пользовательских структур Kaiten от входа на сайт до
конечного отображения Kaiten-источника и целевого Kaiten.

---

## Системы

Kaiten Loader Servise (application)
Frontend (KLS-app)
Backend (KLS-app)
Kaiten (API)

---

## Предусловие

Пользователь заходит на ресурс kaitenloader.ru

---

## Результат

Отображение пользовательских Kaiten-источника и целевого Kaiten во frontend
приложения.

---

## Основной поток

Прямая последовательность шагов.

1. Пользователь заходит на ресурс https://kaitenloader.ru, вызывая метод API GET /main.
2. Frontend отображает интерфейс основной страницы приложения с пустыми формами ввода ссылки и токена Kaiten-источника (далее Kaiten (A)) и целевого Kaiten (далее Kaiten (B)).
    2.1 Если пользователь не ввёл ссылку, токен и не загрузил структуру своего Kaiten (A) в приложение, то формы ввода ссылки и токена для Kaiten (B) заблокированы для ввода, отображаясь серым цветом.
    2.2 Если пользователь решил ввести ссылку или токен для Kaiten (B), не загрузив Kaiten (A), на форме ввода Kaiten (B) появляется уведомление о невозможности заполнить поля, не загрузив структуру Kaiten (A). См. макеты в Figma.
    2.3 Если пользователь вводит ссылку не на Kaiten, появляется уведомление о необходимости ввода ссылки именно на Kaiten. Проверка осуществляется за счёт регулярного выражения со следующей логикой:
   
        > [бесконечное кол-во символов].kaiten.[бесконечное кол-во символов].ru
        > kaiten.[бесконечное кол-во символов].ru
        > [бесконечное кол-во символов].kaiten.[бесконечное кол-во символов].com
        > kaiten.[бесконечное кол-во символов].com
3. Пользователь вводит ссылку и токен от своего Kaiten в формы ввода Kaiten (A).
    3.1 Нажимает кнопку "Загрузить" в окне структуры Kaiten.
   
        > При вводе ссылки на Kaiten в форму ввода Kaiten (A), frontend должен обрезать
        ссылку по следующему примеру:
            > example.kaiten.ru/
4. Frontend заполняет тело запроса и вызывает метод POST https://baseURL/api/v1/kaiten/import.
   Логика заполнения тела запроса:
   
   > ключ role заполняется enum 1;
   > ключ url заполняется доменом пользовательского Kaiten следующего формата: https://example.kaiten.ru/ 
   > ключ token заполняется Bearer token пользовательского Kaiten.
5. Пока frontend ждёт ответ от backend, выполняется анимация спиннер, согласно макетам в Figra.
6. Backend получает тело запроса и вызывает 2 метода API Kaiten, используя пользовательские url и token:
    GET https://example.kaiten.ru/api/latest/spaces - получение данных о пространствах и досках.
    6.1 Из ответа из метода выше, backend, используя id досок (boards.id), вызывает метод:
   
        GET https://example.kaiten.ru/api/latest/boards/{id} - получение данных о колонах доски и карточках на доске.
7. Из данных, полученных из двух методов API Kaiten, backend формирует и возвращает тело ответа к frontend по схеме тела ответа метода POST https://baseURL/api/v1/kaiten/import.
8. Frontend принимает тело ответа от backend и отображает схему пользовательского Kaiten (A) в окне структуры Kaiten, согласно макетам в Figma.
9. После загрузки структуры пользовательского Kaiten (A), формы ввода ссылки и токена для Kaiten (B) становятся доступными к заполнению, изменяя цвет заливки полей ввода.
10. Пользователь заполняет поля ввода для Kaiten (B) ссылкой и токеном. Нажимает "Загрузить" в окне структуры Kaiten.
    
    > При вводе ссылки на Kaiten в форму ввода Kaiten (B), frontend должен обрезать ссылку по следующему примеру:
    > example.kaiten.ru/
11. Frontend заполняет тело запроса и вызывает метод POST https://baseURL/api/v1/kaiten/import.
    Логика заполнения тела запроса:
    
        > ключ role заполняется enum 2;
        > ключ url заполняется доменом пользовательского Kaiten следующего формата: https://example.kaiten.ru/
        > ключ token заполняется Bearer token пользовательского Kaiten.
12. Повторение шагов 5, 6, 7.
13. Frontend принимает тело ответа от backend и отображает схему пользовательского Kaiten (B) в окне структуры Kaiten, согласно макетам в Figma.

Сценарий завершён.
