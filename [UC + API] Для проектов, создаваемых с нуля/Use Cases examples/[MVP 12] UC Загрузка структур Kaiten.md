| 1. Пользователь заходит на ресурс https://base.url.com, вызывая метод API GET /main.<br/>2. Frontend отображает интерфейс основной страницы приложения с пустыми формами ввода ссылки и токена Kaiten-источника (далее Kaiten (A)) и целевого Kaiten (далее Kaiten (B)).<br/><br/>2.1 Если пользователь не ввёл ссылку, токен и не загрузил структуру своего Kaiten (A) в приложение, то формы ввода ссылки и токена для Kaiten (B) **заблокированы для ввода, отображаясь серым цветом**.<br/>2.2 Если пользователь решил ввести ссылку или токен для Kaiten (B), не загрузив Kaiten (A), на форме ввода Kaiten (B) **появляется уведомление о невозможности заполнить поля**, не загрузив структуру Kaiten (A). См. макеты в Figma.<br/>2.3 Если пользователь вводит ссылку **не** на Kaiten, **появляется уведомление о необходимости ввода ссылки именно на Kaiten**. Проверка осуществляется за счёт регулярного выражения со следующей логикой:<br/>> [бесконечное кол-во символов].kaiten.[бесконечное кол-во символов].ru<br/>> kaiten.[бесконечное кол-во символов].ru<br/>> [бесконечное кол-во символов].kaiten.[бесконечное кол-во символов].com<br/>> kaiten.[бесконечное кол-во символов].com<br/><br/>3. Пользователь вводит ссылку и токен от своего Kaiten в формы ввода Kaiten (A). Нажимает кнопку "Загрузить" в окне структуры Kaiten.<br/>> При вводе ссылки на Kaiten в форму ввода Kaiten (A), frontend должен обрезать ссылку по следующему примеру:<br/>> example.kaiten.ru/<br/><br/>4. Frontend заполняет тело запроса и вызывает метод **POST https://baseURL/api/v1/kaiten/import**.<br/>Логика заполнения тела запроса:<br/>> ключ role заполняется **enum 1**;<br/>> ключ **url** заполняется доменом пользовательского Kaiten следующего формата: https://example.kaiten.ru/ <br/>> ключ **token** заполняется Bearer token пользовательского Kaiten.<br/><br/>5. Пока frontend ждёт ответ от backend, выполняется анимация спиннер, согласно макетам в Figra.<br/>6. Backend получает тело запроса и вызывает 2 метода API Kaiten, используя пользовательские url и token:<br/>GET https://example.kaiten.ru/api/latest/spaces - получение данных о пространствах и досках.<br/><br/>Из ответа из метода выше, backend, используя id досок (boards.id), вызывает метод:<br/>GET https://example.kaiten.ru/api/latest/boards/{id} - получение данных о колонах доски и карточках на доске.<br/><br/>7. Из данных, полученных из двух методов API Kaiten, backend формирует и возвращает тело ответа к frontend по схеме тела ответа метода **POST https://baseURL/api/v1/kaiten/import**.<br/>8. Frontend принимает тело ответа от backend и отображает схему пользовательского Kaiten (A) в окне структуры Kaiten, согласно макетам в Figma.<br/>9. После загрузки структуры пользовательского Kaiten (A), формы ввода ссылки и токена для Kaiten (B) **становятся доступными к заполнению, изменяя цвет заливки полей ввода**.<br/>10. Пользователь заполняет поля ввода для Kaiten (B) ссылкой и токеном. Нажимает "Загрузить" в окне структуры Kaiten.<br/>> При вводе ссылки на Kaiten в форму ввода Kaiten (B), frontend должен обрезать ссылку по следующему примеру:<br/>> example.kaiten.ru/<br/><br/>11. Frontend заполняет тело запроса и вызывает метод **POST https://baseURL/api/v1/kaiten/import**.<br/>Логика заполнения тела запроса:<br/>> ключ **role** заполняется **enum 2**;<br/>> ключ **url** заполняется доменом пользовательского Kaiten следующего формата: https://example.kaiten.ru/ <br/>> ключ **token** заполняется Bearer token пользовательского Kaiten.<br/><br/>12. Повторение шагов 5, 6, 7.<br/>13. Frontend принимает тело ответа от backend и отображает схему пользовательского Kaiten (B) в окне структуры Kaiten, согласно макетам в Figma.<br/><br/>Сценарий завершён. |
